# Github 规范

## 项目命令规范

常规项目应该依照其目的遵循添加相关联的前缀或者后缀的命名格式。

### 后缀（Postfix）:

  - `-Notes` 学习笔记/纪录，可以包含学习过程中的源代码（项目或文件）。
  - `-Demo` 展示样例

## Commit Comment规范

通常情况下，采用 `更改动作: 文件（内容）` 的格式。

被允许的更改动作：

  - `add` / `new`: 添加
  - `delete` / `remove`: 删除
  - `update` / `modify`: 更新
  - `move`: 移动目录或文件
  - `rename`: 重命令
  - `refactor`: 重构

### 示例：

```
add: readme.md
delete: cache.tmp
rename: `Role.java` to `UserRole.java`
```
